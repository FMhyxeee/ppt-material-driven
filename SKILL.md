# PPT Material-Driven Skill

按顺序执行，不跳步；每一步都要可审阅、可修改、可回滚。

## 默认风格与防单调预设

默认使用：

- `references/longi-inspired-style.md`（隆基风格参数）
- `references/corporate-minimal-style.md`（版心与结构基线）
- `references/style-presets.md`（防单调增强）

执行规则：

1. 先应用 Corporate Minimal 基线（版心、字体、颜色、留白）。
2. 再从 `style-presets.md` 选择 1 套“增强样式包”（只选一套）。
3. 增强元素占比不超过 20%（避免从“商务”变成“海报”）。

## 0) 输入门槛（先补齐再开工）

先确认 7 项最小输入，缺失则先列 `缺失项清单`：

1. 演示目标（汇报/说服/培训/路演）
2. 听众画像（管理层/客户/技术/学生）
3. 时长与页数上限（例如 15 分钟、12 页）
4. 必须使用的材料（文档/表格/图片）
5. 品牌与风格约束（字体/颜色/语气）
6. 场景类型（商务汇报 / 学术演讲）
7. 截止时间与优先级（先结构还是一次成稿）

## 1) 材料盘点与分级

输出 `材料盘点表`（来源 / 用途 / 建议页位 / 风险）：

- 必须保留：结论、关键数据、合规信息
- 可选增强：案例、背景、补充数据
- 建议删除：重复、跑题、密度过低内容

## 2) 故事线先行

先产出再开页：

1. 一句话主叙事（核心结论）
2. 三段式结构（现状-问题-方案 或 目标-路径-结果）
3. 结尾行动项（希望听众做什么）

要求：每页必须回答“这一页贡献了什么”。

## 3) Mermaid 思维导图（必须先做）

先输出 1 份 `mindmap`，用于结构确认：

- 根节点：核心结论
- 一级分支：3-5 个章节
- 二级分支：每章 2-4 个要点
- 叶子节点：证据素材（数据/案例/图片编号）

未确认导图前，不进入逐页写作。

## 4) Mermaid 到图表映射（必须执行）

输出 `图表转化表`（节点 -> 图表类型 -> 页面类型 -> 备注）：

- 层级关系 -> 树状/组织结构图
- 顺序关系 -> 流程图
- 对比关系 -> 双栏对照/对比表
- 趋势关系 -> 折线/柱状图
- 占比关系 -> 条形占比/环形图（谨慎）

规则：图表服务结论，不为“好看”替换清晰表达。


## 4.5) 商务/学术分支（必须声明）

在输出逐页蓝图前，先给 `场景适配声明`：

- 当前场景：商务汇报 或 学术演讲
- 商务模式：每章结尾包含行动项（Next Step）
- 学术模式：每章结尾包含证据/局限/展望
- 动画策略：商务可少量章节过渡；学术默认极简动画`r`n`r`n## 5) 页面骨架（8 种商务母版）

优先使用以下页面类型保证一致性：

1. 封面
2. 目录
3. 章节页
4. 左文右图（40/60）
5. 三卡片
6. 对比页
7. 流程页
8. 总结页

## 6) 逐页蓝图（Slide Blueprint）

按页输出，每页必须含：

- 页码 + 页面类型
- 标题（<=14 字，结论先行）
- 一句话 Takeaway
- 主体文案（3-5 条短句）
- 视觉建议（图表/图片/版式）
- 素材映射（材料来源、图片编号）
- 讲者备注（30-90 秒）

信息不足时标注 `TODO(需用户补充)`，禁止编造事实。

## 7) 图片与素材规则

逐张给出：

- 图片编号与来源（可追溯）
- 推荐页码与用途
- 裁切建议（16:9 安全区、主体位置）
- 覆盖文案建议（如需）
- 风险提示（清晰度/版权/歧义）

看不清就写 `需高清原图`。

素材来源优先使用 `references/asset-sources.md`，并在交付里附来源说明。

## 8) 精修步骤（防单调但不花哨）

在内容完成后执行 1 轮精修：

1. 统一基线：字体层级、页边距、栅格、留白、页脚系统。
2. 选 1 套样式增强包（见 `style-presets.md`）。
3. 仅在关键页加轻量视觉元素：
   - 章节页：细分隔线或色块锚点
   - 对比页：强调色只给“结论列”
   - 数据页：高亮关键数值 + 注释标签
4. 执行“单调度检查”：连续 3 页不得使用同一版式构图。

## 9) 每 3 页做一次结构回看

每完成 3 页输出：

- 已覆盖叙事节点
- 尚未覆盖但必须出现的要点
- 下一批页面计划（页码范围 + 目标）

## 10) 交付格式（可直接用于PPT）

最终输出必须包含：

1. `总目录`
2. `逐页内容`
3. `图片放置清单`
4. `统一风格规则`
5. `终检清单`
6. `Mermaid 源码`
7. `图表转化表`
8. `素材来源清单`（链接 + 许可说明）
9. `场景适配说明`（商务/学术差异化策略）

## 11) 质量门禁

逐项检查：

- 标题是否结论先行
- 单页是否只讲一件事
- 数据是否有来源与时间
- Mermaid 节点是否全部落到页面/图表
- 图片是否服务结论
- 术语与口径是否全篇一致
- 页数与时长是否匹配（约 1 页/1-2 分钟）
- 增强样式是否克制（强调色是否超量、装饰是否喧宾夺主）

不满足时先给 `修订建议列表`，再输出修订版。

## 输出约束

- 默认中文简体；用户指定英文时切换。
- 先结构、后文案、再精修。
- 优先短句与项目符号，减少大段文字。
- 避免空泛词堆叠。
